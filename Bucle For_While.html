Control de LEDs Secuenciales con Arduino

Introducción Código Descripción Simulación Aplicaciones Conclusiones

1. Introducción

Este informe describe el funcionamiento de un programa desarrollado en C++ para Arduino. El código controla el encendido y apagado secuencial de siete LEDs conectados a pines digitales, generando un efecto visual ordenado y repetitivo.

2. Código Utilizado

// C++ code
int leds[] = {3, 4, 5, 6, 7, 8, 9};
int numLeds = 7;

void setup() {
    for (int i = 0; i < numLeds; i++) {
        pinMode(leds[i], OUTPUT);
    }
}

void loop() {
    int i = 0;
    while (i < numLeds) {
        encender_apagar(i);
        i++;
    }
}

void encender_apagar(int i) {
    digitalWrite(leds[i], HIGH);
    delay(1000);
    digitalWrite(leds[i], LOW);
    delay(1000);
}

3. Descripción del Funcionamiento

El programa utiliza un arreglo de pines para definir los LEDs conectados. En el setup(), se configuran como salidas. En el loop(), se recorre el arreglo de manera secuencial llamando a la función encender_apagar(), la cual enciende un LED durante 1 segundo, lo apaga otro segundo y luego continúa con el siguiente. Este proceso se repite indefinidamente.

4. Simulación en Tinkercad

En esta simulación realizada en Tinkercad, se observa cómo los LEDs se encienden y apagan en forma secuencial, reproduciendo el comportamiento descrito en el código. Esta herramienta permite validar el diseño de manera interactiva antes de montarlo físicamente.

5. Aplicaciones Prácticas

  • Señalización luminosa (ejemplo: semáforos).
  • Efectos decorativos (luces navideñas, juegos de luces).
  • Aprendizaje básico de control secuencial en Arduino.
  • Simulación de estados en sistemas electrónicos.

6. Conclusiones

Este proyecto demuestra una forma sencilla y eficiente de controlar múltiples LEDs mediante programación en Arduino. El uso de arreglos y funciones facilita la organización del código y su escalabilidad. Los retardos de un segundo permiten visualizar claramente el efecto secuencial, aunque pueden ajustarse según la necesidad del proyecto.
